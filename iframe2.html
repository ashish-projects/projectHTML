I had the same problem with a link inside an iFrame. I was using the absolute url ("http://google.com") and ("https://google.com"). I finally got it to work by using the relative path to the page ("././index.html/"). My guess is that for security reasons "foreign" links using https or http are "disarmed" now in iFrames, even if they are on the same domain. Because the link (absolute url) works when not using https or http.
