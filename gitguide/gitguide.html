This Section Covers Git

Day : Check if git is installed

git : this will provide the information

Day : Install Git

Install git from the link:

Things to check while installing:

Day 1: Check Git Version

git --version

Day : Git Help

git help

git help command : eg: git help commit

Day : Config Username and Email

git config --global user.name "nameOfUser" : configures Name

git config --global user.email "emailOfUser" : configures email

Day : Check Username And Email Configured to Git

git config user.name : provides name configured with git

git config user.email : provides email configured with git

git --list : provides all configured details

Day : Check if repo/project is Git initalized

git status

Day : Initialize Git Repo

git status : check if it is or isn't initialized

git init

DAY : Check Status

git status : check status of file in working area and staging area, red and green colored resp

DAY : Adding a File

git status : check status

git add filename : adds a single file

git add . : adds all files

DAY : Commiting Changes

git status : check status

git commit -m "commit message"

DAY : Checking Commits

git log : provides all commits

git log --author="contributor Name"

DAY : Checking Difference

git diff : provides difference between working copy and repo on git

git diff --staged : difference between staged copy and repo

DAY : Removing File

git rm "filename" : file can be deleted from git bash too

DAY : Renaming File

git mv "filename1" "filename2" : file can be deleted from git bash too

git mv "filename1" "folder/filename2" : file can be deleted from git bash too

DAY : Commiting Without Adding

git commit -am "commit message without adding to staged area"

DAY : Getting Backup of any File not added/commited

git status

git checkout --filename

DAY : Unstagging a File

git reset Head

DAY : Getting old version after commiting

git log

git checkout "first few digits of branch

git status

DAY : Deleting a Branch

git branch -D branchname
