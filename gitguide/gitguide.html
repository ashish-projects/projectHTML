This Section Covers Git

Day : Check if git is installed

git

Day : Install Git

Install git from the link depending on operating system

Day: Check Git Version

git --version

Day : Git Help

git help

git help command : eg: git help commit

Day : Config Username and Email

git config --global user.name "nameOfUser" : configures Name

git config --global user.email "emailOfUser" : configures email

Day : Check Username And Email Configured to Git

git config user.name : provides name configured with git

git config user.email : provides email configured with git

git config --list : provides all configured details

Day : Check if repo/project is Git initalized

git status

Day : Initialize Git Repo

git status : check if it is or isn't initialized

git init

DAY : Check Status

git status : check status of file in working area and staging area, red and green colored resp

DAY : Adding a File

After adding or modifying a file in a project on local machine

git status : check status

git add filename : adds a single file

git add . : adds all files in a folder

DAY : Commiting Changes

git status : check status

git commit -m "commit message"

DAY : Commiting Without Adding

git commit -am "commit message without adding to staged area, to be used only when modifying file or creating one"

DAY : Checking Commits

git log : provides all commits

git log --author="contributor Name"

git log --pretty=oneline

git log --pretty=oneline (anyNumber) e.g git log ==pretty=oneline -3

DAY : Checking Difference

git diff : provides difference between working area and commited area

git diff --staged : difference between staged area and commited area

DAY : Removing File

Unlike modifying or creating a file, when you delete a file

git rm "filename" : file can be deleted from git bash too using the same command

DAY : Renaming File

git mv "filename1" "filename2" : file can be deleted from git bash too

git mv "filename1" "folder/filename2" : file can be deleted from git bash too

DAY : Getting Backup of any File not added/commited

git status

git checkout --filename

DAY : Unstagging a File

git reset Head

DAY : Getting old version after commiting

git log

git checkout "first few digits of branch

git status

Check Branches on the git

git branch

Creating a branch

git branch : check branches on the git

git branch branchname : created branch with name provided

git checkout -b branchname : creates and checkout that branch

git branch : check branches on the git

Change branch

git checkout branchname

DAY : Deleting a Branch

git branch : check branches on the git

git branch -D branchname

git branch : check branches on the git

Merging two branches

git merge UpdatedBranch : After Going to the branch that is not update

DAY: Stopping git from unTracking some files

Create file .gitignore and put the file name on the same
