This Section Covers Most Common Flow when working on forked Repo

  1. Fork a GitHub repository.From the github
  2. Clone the forked repository to your local git clone url(your github url of cloned repo, not theirs)
  3. Add a Git remote for the original repository.git remote add upstream url (forked projects URL, not yours)
  4. CHeck branches git remote -v
  5. Create a feature branch in which to place your changes.git checkout -b "newBranchName">
  6. Make your changes to the new branch simply as on standalone git git add * or filename
  7. Commit the changes to the branch simply as on standalone git git commit -m "message"
  8. Push the branch to GitHub, this will create same branch on your github too git push origin "newBranchName"
  9. Open a pull request from the new branch to the original repo. From the github which by now will prompt you to create pull request
  10. Update your local project with upstream.git pull upstream master
  11. Delete the branch just created : "newBranchName"git branch -d "newBranchName"
  12. Update master branch in forked repo : git push origin master
  13. Delete the new branch created on forked version too git push --delete origin "newBranchName"
